# code-health-auditor

## SUMMARY
## Dashboard code health audit — findings

**Scale.** `dashboard/server.py` 1981 lines, `dashboard/index.html` 4145 lines (1078 CSS / ~180 markup / ~2870 JS). Both are single-file accretions of ~30 PR-sized slices; line-range maps for every responsibility are in key_facts.

**server.py is 6 programs in one file**: (1) hand-edited PIPELINE topology spec (39-201), (2) filesystem discovery (skills/agents/hooks/ADRs/body-grep edges, 252-658), (3) a full reimplementation of the audit-meta + audit-subagents check suites (749-1125) — which is also the CI CHECK 9 import surface, (4) event-log query engine with reverse tail-seek (1147-1174, 1314-1504), (5) gh-CLI workitems fetcher (1181-1282), (6) README/mermaid generator (1599-1974). The HTTP handler (1289-1573) is thin routing over these.

**index.html concentrates risk in two mega-functions**: `renderTopologyGraph` (1536-2000, 465 lines, three mode branches with the BFS level-assigner copy-pasted twice — `bfsEnqueue`/`bfsEnqueue2`, both now carrying the cycle-cap that had to be shipped as TWO separate crash fixes #637/#638) and `buildGraphData` (2658-3105, 448 lines). Five separate side-panel DOM structures with four independent open/close implementations; the extra-fields HTML block is pasted 4x inside `viewFile`; the agent start/complete pairing algorithm exists twice; three scattered time helpers plus inline slicing.

**Dead code is substantial and load-bearing-looking**: the `/api/events` SSE endpoint has zero clients (no `EventSource` anywhere — the Live tab is 30s polling), yet the docstring advertises it and its implementation re-reads the whole log every second per connection. `_dispatchMapFromSpec`/`_orchestratorChildrenFromSpec` (the old children[]-tree renderers) are dead, which makes PIPELINE's entire `children[]` mechanism consumed-by-nobody while its comment claims it 'drives /api/pipeline topology'. `toggleDetail`, `_grep_count`, `_grep_fixed` dead; the `'m:'` lookupId ternary (1981) is the confirmed #631 no-op so measured-node clicks 404; ~45 CSS classes (~170 lines, mostly 502-669) are orphans of deleted live-stream renderers.

**Why the three commissioning symptoms happen, from this code's side**: (1) *declared ≠ measured by construction* — `deriveMeasuredEdges` emits only `orchestrator→skill` and `lastSkill→agent` pairs while the declared spec routes through virtual artifacts (prd_issue/pr/merge…) that events can never name; at most ~7 of 36 declared edges can ever match, orchestrator→skill is always 'unexpected red', and measured mode uses a different (flat) layout. (2) *hooks 'never fired'* — verified live: all 7 inline hooks collapse to one display name `log-event` with a shared aggregate count, and the per-hook beacons (`pre-tool-bash` etc.) write to per-worktree `.claude/logs` without mkdir, silently dropping fires the main-repo-reading dashboard never sees (`fires:0` despite constant use). (3) *Live tab* — `_serve_runs` assumes a session's lines are contiguous and truncates at the first interleave, yet `log-event.sh` deliberately funnels ALL worktree sessions into one file so interleaving is the norm during /ship; and the log itself was, until a manual rename **today**, 3648 lines dominated by synthetic/demo/test sessions and 2334 empty-session_id events.

**Fixture leakage**: the archive `workflow-events.jsonl.synthetic-archive-20260610` proves QA/dev sessions wrote fixture logs (`sess-test-001`, `demo-build-7a3f`, `demo-ship-2e9c`, `test-123`…) straight into live log paths via bash heredocs — and those heredoc commands were themselves captured as real `bash_complete` events. Nothing ever entered git (gitignored; `git log -S` clean). **There is no guard today**: no schema validation, no test-session namespace, no reject path; the only separator that exists (`LOGFILE` env override) is used solely to route subagent-edit nudges to a different file.

**Decomposition (migration-risk-weighted)**: CHECK 9 does `sys.path.insert('dashboard'); import server` and calls exactly `server.check_docs1_*..check_docs10_*` — so any server split must keep `server.py` as a re-exporting facade over flat sibling modules (pipeline_spec / discovery / health / events / workitems / readme_gen), stdlib-only; a single-file section-reorder + dead-code purge is the acceptable cheaper fallback. The client split has one hidden blocker: the HTTP server has **no static-file route** (only `/` and `/api/*`), so extracting `js/*.js` first requires a whitelisted static route reusing `/api/file`'s traversal guard. Vendoring the two CDN scripts (vis-network, marked) would make the 'zero-dependency local tool' claim actually true offline. Full prioritized list in opportunities.

## DEFECTS
- [critical] dashboard/index.html:1406-1449 (deriveMeasuredEdges) vs server.py:156-200 (PIPELINE __edges__): Declared and measured topologies use structurally disjoint edge vocabularies, so 'declared == measured' can never converge by construction. Measured edges are only ever {orchestrator->skill, lastSkill->agent}; the declared spine routes through virtual artifact nodes (prd_issue, slice_issues, pr, merge — 12 of 36 declared edges) that no event can ever reference, and orchestrator->skill is never declared (spec uses U1->build/grill_me). Result: in 'both' mode every measured orchestrator->skill edge renders permanently red ('unexpected'), most spine edges render permanently gray ('never observed'), and at most ~7 of 36 declared edges (skill->critic pairs like to-prd::prd-critic, qa-plan::qa-tester) can ever turn green. Measured mode also uses a flat 3-level layout vs declared BFS-depth layout, so the two graphs look 'totally different' even with perfect data. This is the root cause of audit symptom (1).
- [critical] .claude/logs/ (runtime) + .claude/hooks/log-event.sh + qa-proof/*.py: No guard separates test/synthetic data from the production event log. Archived log workflow-events.jsonl.synthetic-archive-20260610 (3648 lines) contains demo sessions (demo-build-7a3f x25, demo-ship-2e9c x14), test ids (abc, test-123, verify-123, manual-test-0609, sess-test-001), junk event types (test_hook_fire, manual_probe, no_sid_test, canonical-resolution-test, reviewer-test), and 2334 events with EMPTY session_id (all lumped into one 'unknown' pseudo-session by _serve_runs). QA/dev sessions wrote fixtures directly into the live log path via bash heredocs, and those very heredoc commands were themselves logged as bash_complete events (double pollution). The only 'remediation' was a manual rename today (2026-06-10). Remaining quasi-guards — LOGFILE filename override and .gitignore — do not validate, tag, or namespace anything.
- [major] dashboard/server.py:1356-1364 and 1417-1444 (_serve_runs): Session grouping assumes a session's lines are CONTIGUOUS in the log. The ?session= branch stops collecting at the first non-matching line after the target sid is seen ('elif sid_seen: break'); the metadata branch stops counting a session's events once n distinct sids are collected. But log-event.sh deliberately funnels ALL worktree sessions into the single main-repo log, so concurrent orchestrator+subagent sessions interleave as the norm during /ship — interleaved sessions are silently truncated (wrong event_count, wrong first_ts, partial event lists). Plausible major contributor to audit symptom (3) 'Live tab never worked'.
- [major] dashboard/server.py:520-521 (discover_hooks) + .claude/hooks/log-event.sh:69 + _read_hook_name:389-391: Hook-fire telemetry join is broken two ways (empirically verified by running discover_hooks): (a) all 7 inline logger hooks pipe through log-event.sh, and _read_hook_name's .sh regex matches 'hooks/log-event.sh' first, so all 7 display as the SAME name 'log-event' with the SAME aggregate fire count — per-hook fire counts for agent_start/agent_complete/bash/skill_invoke/grill_qa/session_stop are unknowable from the UI; (b) the inline-pattern name branches in _read_hook_name (lines 393-410) and _read_hook_description are consequently dead code.
- [major] .claude/hooks/pre-tool-bash.sh:9, pre-tool-edit.sh:25, session-start.sh:9, user-prompt-submit.sh:9 (beacon lines): Per-hook fire beacons write to ${CLAUDE_PROJECT_DIR}/.claude/logs/hook-fires.jsonl WITHOUT mkdir -p and with '|| true'. In a worktree session the path is the worktree's .claude/logs which typically does not exist (verified: current worktree has none), so beacons are silently dropped; meanwhile the dashboard reads only the main repo's hook-fires.jsonl. Verified live: pre-tool-bash shows fires:0 despite constant Bash use this session. Root cause of audit symptom (2) 'some hooks have never fired' being misreported. Note log-event.sh uses a DIFFERENT (correct) git-common-dir resolution — two divergent path-resolution strategies in the same hook suite.
- [major] dashboard/index.html:1731-1791 (declared) vs 1850-1933 (both): BFS level-assignment exists exactly 2 times, copy-pasted: bfsEnqueue/bfsAdj/queue-walk vs bfsEnqueue2/bfsAdj2/queue-walk2, plus duplicated node-placement and edge-render loops (~60 lines). Both copies now carry the cycle-cap guard — but only because the same OOM bug had to be fixed twice (PR #637 for declared, PR #638 for both), proving the duplication already caused a shipped defect.
- [major] dashboard/server.py:1458-1504 (_serve_sse) + /api/events route at 1545-1546: The SSE endpoint has ZERO clients — no EventSource and no '/api/events' string anywhere in index.html (Live tab polls /api/runs on a 30s setInterval instead). - [minor] dashboard/index.html:2372-2374 (escHtml) + onclick attribute construction at 2075, 2185: escHtml does not escape single quotes, and click handlers are built as onclick="viewFile('PATH','NAME')" with single-quoted JS args inside double-quoted attributes — a value containing a single quote breaks out of the string. ## OPPORTUNITIES
- Fix the declared-vs-measured comparison at the vocabulary level before any visual work: either (a) extend the measured deriver to synthesize artifact transitions (e.g. agent_complete of reviewer with VERDICT:APPROVE => pr->reviewer->merge) so it speaks the __edges__ vocabulary, or (b) add a 'comparable projection' of PIPELINE (skill->agent dispatch edges only, orchestrator-rooted) and diff measured against THAT, rendering artifact edges as context-only. Today the drift view is structurally incapable of going green and should not ship as a verification surface.
- Extract one topology BFS helper (assignLevels(specEdges) returning {levels, adj}) and one branch-shared 'renderDeclaredBase()' used by both declared and both modes — eliminates the duplicated cycle-cap code that already produced two separate crash fixes (#637/#638). ~120 lines collapse to ~50.
- Unify the panel layer: one showPanel(panelId, {title, fields, bodyHtml}) + one renderExtraFields(fields) helper replaces the 4 copy-pasted extra-fields blocks in viewFile, the duplicated no-source branch in the topology click handler, and the parallel docs/as health-detail implementations (already string-parameterized — trivially mergeable).
- server.py decomposition with CHECK 9 survival: keep dashboard/server.py as a facade. Lowest-risk split: sibling flat modules in dashboard/ — pipeline_spec.py (PIPELINE + mermaid renderer), discovery.py (discover_* + hook helpers + telemetry), health.py (check_docs1-10 + AS-* + enrichment), events.py (_iter_lines_reversed + runs logic), workitems.py, readme_gen.py — with server.py doing 'from health import *' style re-exports so 'import server; server.check_docs1_...' keeps working unchanged (CHECK 9 does sys.path.insert(dashboard) so flat sibling imports resolve). Alternative if splitting is judged too heavy for a zero-dep local tool: single-file reorder into banner-labeled sections in dependency order (spec -> discovery -> health -> events -> http -> readme-gen) and move main() to the bottom; either way delete _grep_count/_grep_fixed/_serve_sse first.
- Delete the dead /api/events SSE endpoint + docstring line, or replace polling with it properly (incremental tail via _iter_lines_reversed, not readlines-per-second). Don't keep both a dead push channel and a poll channel.
- index.html decomposition: add a safe static route (reuse /api/file's traversal guard, whitelist dashboard/js/ + dashboard/css/), then extract js/topology.js (1285-2000), js/viewer.js (2030-2374: hooks section, viewFile, summarize, escHtml), js/health.js (2379-2524), js/live.js (2537-3674), js/context-panel.js (3684-4135), css/dashboard.css. - Fix the telemetry join: log-event.sh should beacon the EVENT name it just routed ({"hook":"agent_start"}…) instead of 'log-event', or discover_hooks should key inline hooks by their logged event type; and all beacon writers should adopt log-event.sh's git-common-dir resolution + mkdir -p so worktree sessions stop silently dropping fires. This single fix makes the 'never fired' badge truthful (audit symptom 2).
- Cache /api/health (mtime-keyed or 30s TTL like workitems) and parse each SKILL.md once per request instead of per-check; make cascade_finder_summary check returncode or delete the subprocess call (it ignores its result today).
