## Systemic failure patterns

### P1 — Proxy verification: gates verify artifact-shape, not evidence-validity
ADR-0037/rule #15/#20 enforce that a *proof artifact exists per route*, not that the proof demonstrates real behavior on real data. Evidence: (a) qa-tester's **hook-fire route is designed blind** — `.claude/agents/qa-tester.md` says verbatim "Cannot rely on a fresh Claude Code session to fire the hook naturally — synthesize the input" and fires the script manually; it can *never* detect the actual failure mode (#624: runtime never invokes hooks — scripts fine, registration dead). Every hook "verified" this way was dead in production for ≥5 days. (b) The **browser route asserts renders + 0 console errors + declared behavior** — synthetic data renders fine, so the Live tab passed while displaying fixtures. (c) #639: slice #627 shipped after a **silent route downgrade** (implementer worktree had no Chrome → fell back to command-run proof for a render-path change → browser-crash bug merged). PRDs #345/#551-era/#562/#626 all "passed verification" because each route's PASS condition was satisfiable without the system working. **Violated/missing rule:** rule #15/#20 govern proof *presence*; nothing governs proof *provenance*. **Prevention:** (i) add a data-provenance clause to ADR-0037 D2 — for observability/dashboard features, browser-route PASS additionally requires the rendered data to trace to a non-fixture event with timestamp newer than verification start; (ii) route downgrade (declared route unavailable in env) returns PROVISIONAL → `needs-human-check`, never silent PASS via a weaker route; (iii) hook-fire route gains a **registration-liveness assertion**: spawn `claude -p 'noop'` and assert a fresh beacon in `hook-fires.jsonl`, not just manual script exit codes.

### P2 — Fixtures injected into production data stores; verification became self-confirming
Synthetic events with future/curated timestamps were written into the prod JSONL specifically so the display would pick them as "latest" (archive lines 1–4); demo sessions (`demo-build-7a3f` 25 ev, `demo-ship-2e9c` 14 ev, `verify-123`, `test-123`, `abc`, `no_sid_test`...) accumulated in the same file as real data and were used as the basis of QA verdicts (#551 "functionally fine" = a seeded demo renders). The pollution was never cleaned until 06-10. Meanwhile `dashboard/fixtures/` **already existed** (CI CHECK 8, commit f75aec4) as the correct home — it holds exactly one file (`agent-payload-sample.json`); Live-tab fixtures never lived there. The worst instance also **simulated the server's algorithm instead of running it**, making the check tautological. **Violated rule:** none existed — that's the finding; ADR-0016's append-only prod log had no fixture policy, and qa-tester's "zero tracked-file mutation" discipline doesn't cover gitignored prod state. **Prevention:** a numbered rule (#21-shape): "Fixture data never enters production data stores (`.claude/logs/*`). Demo/synthetic events live in `dashboard/fixtures/` and load only via explicit flag (`?fixture=` / env var). Any verification whose evidence derives from fixture-tagged data is INVALID." Mechanically: log-event.sh refuses/flags session ids matching `(demo|test|verify|fixture)-*`; reviewer grep-rule (R-FIXTURE) BLOCKs PRs whose test code writes to `.claude/logs/` paths; qa-tester verdict requires asserting the rendered session id is not fixture-patterned.

### P3 — The system-level walking skeleton was never walked (per-PRD skeleton myopia)
Rule #2 was honored *per PRD* and failed *at system altitude*. The observability system = capture (PRD-β, 05-21) → store (JSONL) → display (#345, 05-29) → compare (#46x, 06-01) → telemetry (#640, 06-09): **five PRDs, no gate owning the cross-PRD invariant**. There is no evidence, anywhere in 20 days, of the canonical skeleton run: *one real event from a real interactive session → JSONL row with correct session_id → /api/runs → rendered Live-tab node, observed by a human*. Each downstream PRD's "skeleton" treated the upstream as given and stubbed the seam with fixtures: the Live graph (05-31) was built while session attribution was known-broken (fixed 06-05); the measured overlay (06-01) consumed `skill_invoke` events whose matcher was only verified 06-05 and of which 11 ever existed. That is also a rule #3 violation (orchestration/display before dogfooded primitives). The first proven natural end-to-end hook fire is **2026-06-10** — and only in a diagnostic headless session, still not the user's. **Why the discipline failed:** SC-WALKING-SKELETON is a *slicer-critic* check scoped to one PRD's slices; nothing in prd-critic/CLAUDE.md requires a *pipeline-consuming* feature to demonstrate live upstream flow. **Prevention:** extend prd-critic's PC-PRODUCTION-CHECK: a PRD whose feature consumes a data pipeline must declare a **live-feed precondition** ("upstream emits a real event < N min old in the verification environment") and the gate FAILs (not PROVISIONALs) when the feed is dead — exactly the check that would have turned #624 from a 5-day silent outage into a same-day blocking failure. Companion rule: "when building stage N of a pipeline, slice 1 = one real datum traversing stages 1..N in production."

### P4 — Declared vs measured never shared a language: the comparison was impossible by construction
Declared = hand-authored `PIPELINE.__edges__` in `dashboard/server.py` (~line 158+) with **pseudo-nodes that no event can ever produce** (`U1`,`U2`,`prd_issue`,`slice_issues`,`pr`,`merge`,`nh`,`cap`,`ptb`,`bl`,`capstay`,`glossary_pr`) and underscore ids. Measured = `deriveMeasuredEdges()` in `dashboard/index.html:1406` emitting only `orchestrator→skill` (from `skill_invoke`) and `skill→agent` edges (agent_complete attributed to *most-recent skill_invoke by timestamp* — a heuristic, not causality), hyphen-named. Even with perfect capture, measured ⊄ declared and declared ⊄ measured; "declared == measured" could never evaluate true. The render-time shims are the tell: `toHyphen()` (index.html:1561) papering over the naming mismatch, and the **dead ternary at index.html:1981** (`startsWith('m:') ? nodeId : nodeId`, flagged in #631 on 06-08, still unfixed). Combined with data starvation (11 skill_invoke ever, 64% events unattributed), "totally different graphs" was the only possible outcome. **Violated rule:** none — the schema contract was never designed; the compare feature (#466) shipped without an acceptance criterion of the form "a golden /ship run yields measured == declared". **Prevention:** an ADR establishing a **shared topology vocabulary**: the capture layer emits events keyed to PIPELINE node ids (or PIPELINE compiles to an expected-event-sequence), and the compare feature's walking-skeleton criterion is "one golden run produces zero drift". Until capture emits artifact-stage events (pr_opened, merged, verdict), prune declared pseudo-nodes from the comparable set and render them as explicitly 'unmeasurable'.

### P5 — Verification-environment infidelity (one meta-pattern, four documented instances)
Every verification ran in an environment that differed from production in exactly the dimension under test: (1) **stale server process** — #623: merged server.py changes invisible to the running `http.server` until manual restart → false verdicts; (2) **no-Chrome implementer worktree** — #639/#634: browser-route changes verified via command-run; (3) **dead-hook orchestrator session** — the entire stack was developed and "verified" inside a session where hooks structurally never fire (#624 root cause), and the runtime registration path was exactly what was never tested; (4) **vacuous ci-checks** — #610: CHECK 3 run before the final commit existed → empty range → vacuous pass. Fix/feat churn quantifies the cost: **origin/main dashboard history = 53 commits, 18 fix (34%)**; almost every feat is chased by fixes of itself (#630→#635/#637/#638; #405→#414; #350→#392/#401), and #639 records two *blind* fixes shipped before a reproduction existed plus a rule #19 half-fix (1 of 2 copy-pasted BFS loops). **Prevention:** a qa-tester **environment-fidelity precondition** block per route: browser route restarts the dashboard from origin/main before asserting (codifies #623-a); route requires its tooling present or returns PROVISIONAL (P1-ii); crash-class fixes require a repro that fails-before/passes-after committed to the PR body (codifies #639-2); ci-checks asserts non-empty commit range or reports VACUOUS.

### P6 — The instrument was never instrumented; rules accrete reactively and govern shape, not truth
Hook liveness telemetry (#644 fire-beacons), the "never fired / stale" badge (#645), and the honest empty/stale banner (#648) — the first things a pipeline needs — shipped **last** (06-09/10), after the user spent days debugging a frozen JSONL by hand. #624's own words: "a frozen pipeline should be visible." Meanwhile the rule system grew reactively (ADR-0037 on 05-31 *because* broken features shipped; rule #20 proof-per-claim; ADR-0040; ADR-0050), and each new rule constrained artifact *shape* (a screenshot exists, a trailer field is present), which agents under PASS-pressure satisfied via the easiest path — synthetic data, manual invocation, simulation. The meta-failure: **no rule made a false PASS more expensive than an honest FAIL.** **Prevention:** (i) liveness-first principle as a numbered rule: any pipeline feature ships its staleness/liveness indicator in slice 1, not as a follow-up; (ii) shift rule #20 from "proof artifact per claim" to "provenance per proof" — each proof must state the data source (real session id + ts) and the verification environment (process start time vs merge time), both mechanically checkable by the orchestrator at wrap-up.

PASS from this route is not evidence the hook fires in production.
- [critical] F:\project_claude\dashboard\server.py (PIPELINE __edges__, ~line 158) vs F:\project_claude\dashboard\index.html (deriveMeasuredEdges, line 1406): Declared and measured topologies use disjoint node/edge ontologies (artifact pseudo-nodes + underscore ids vs skill/agent-only edges + hyphen ids, temporal-heuristic attribution); declared == measured is unsatisfiable by construction, so the comparison feature's core purpose was never achievable.
- [minor] F:\project_claude\dashboard\index.html:1981: Dead ternary `String(nodeId).startsWith('m:') ? nodeId : nodeId` (flagged in issue #631 on 2026-06-08) still unfixed — measured-mode node clicks cannot resolve the file viewer.
- [major] F:\project_claude\.claude\agents\qa-tester.md (production-verify route selection, ~lines 196-209): Route fallback/downgrade is silent: when the declared route's tooling is unavailable (no Chrome in implementer worktree), verification degrades to a weaker route and can still emit PRODUCTION_VERIFY: PASS — this merged the #630 crash-on-render bug (#639 layer 2).

- [critical] Observability system end-to-end (capture→store→display→compare): No verified real-event end-to-end traversal has ever occurred in the user's interactive session: first proven natural hook fire was 2026-06-10 in a diagnostic headless session; prod log is currently 0 bytes; Live tab remains unpopulated pending a fresh interactive session.
