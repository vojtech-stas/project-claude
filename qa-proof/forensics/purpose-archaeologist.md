## SUMMARY
# Purpose archaeology: workflow-observability stack — intended vs built

## (a) What the repo is FOR
`project-claude` is a clone-as-template starter for AI-coded projects that replicates "a **senior engineer overseeing a small team of developers** — but with AI agents instead of humans" (README.md L4). The delivery machine is an autonomous PRD→Slice→PR pipeline (ADR-0003) with exactly two human touch-points (`/grill-me` at start, `/qa-plan` at end), adversarial critics gating every generation stage, and merge autonomy in the middle. The observability layer (hooks → JSONL event log → dashboard) exists for one stated reason: the owner could not tell whether the autonomous machinery was actually running as designed — ADR-0016 ships explicitly to close the user-flagged gap *"I am not sure if the agents and skills are firing correctly."* The whole stack's purpose is **closing the loop between declared workflow and observed runtime**.

## (b) Dashboard role per PRD #345 + ADR-0039 — and the divergence
**Intended.** PRD #345: a "unified visual control panel" with three tabs — Architecture (auto-discovered component graph), **Live** ("what's firing right now during a `/ship` run (which subagent, which hook, which bash call), in real time"), Health (audit verdicts). ADR-0039 then made the topology single-source: D1 — one machine-readable spec declaring "`orchestrator → skills → the subagents each dispatches` (the `DISPATCH_MAP` content, promoted to the single source)"; D2 — README mermaid AND dashboard topology both *generated* from it; D3 — a **measured overlay** "derived from real `skill_invoke`→`agent_complete` events", with comparison semantics defined at the **dispatch-edge** level: flag "a skill that dispatched an agent the spec doesn't list, or vice-versa." The intended comparison is dispatch-graph vs dispatch-graph — same node kind, same edge kind.

**Built.** `dashboard/server.py` `PIPELINE` dict contains **two different declared topologies**: `children[]` (the dispatch topology — exactly ADR-0039 D1's shape, drives `/api/pipeline`) and `__edges__` (a narrative pipeline-flow clone of the README mermaid, with human nodes `U1`/`U2` and artifact nodes `prd_issue`/`slice_issues`/`pr`/`merge`/`cap`/`ptb`, underscore-named). PRD #626/slice #627 — fixing "topology looks wrong" — switched the rendered *declared* graph to `__edges__`. But `deriveMeasuredEdges()` (index.html L1406) still produces **dispatch-shaped** edges: `orchestrator→skill` per `skill_invoke`, plus `lastSkill→agent` per `agent_complete` via a most-recent-skill temporal heuristic, over events merged from up to 20 sessions sorted globally by timestamp (cross-session and fixture pollution included). The `both` mode (L1837+) then compares dispatch edges against narrative-flow edges. **This is a category error that guarantees "totally different results" even when runtime matches intent**: declared spine edges (`slice_issues→implementer`, `implementer→pr`, `pr→reviewer`, `merge→qa_plan`) are unobservable by construction (artifacts emit no events); measured edges (`orchestrator→ship`, `ship→implementer`, `ship→reviewer`) appear nowhere in `__edges__` by construction, so they render as red "unexpected" drift. Secondary defects: underscore-vs-hyphen name forms only partially bridged; and the measured side was fed by a log that was first largely synthetic, then truncated to zero — so the overlay never once measured a real organic run against the spec.

## (c) ADR-0016 event-log intent — and the divergence
**Intended.** Append-only JSONL at `.claude/logs/workflow-events.jsonl`, **hook-delivered only** (no in-skill instrumentation), three event types in slice 1 (`agent_complete`, `bash_complete`, `session_stop`), grep-friendly. Consumers: any future session reconstructing what happened (extends ADR-0006 D2 session continuity), future audit-meta tooling, a possible query skill. Hooks are scoped by ADR-0015 D2 to logging/validation/notification only.

**Built.** The schema grew (per PRDs #424/#467) to `agent_start`, `skill_invoke`, `grill_qa` etc., delivered via **7 inline jq one-liner commands in `.claude/settings.json`** piping into `.claude/hooks/log-event.sh`. The capture layer is **dead**: issue #624 verified zero new events during a live test (3647→3647, log frozen at 2026-06-05 14:16). Today's empirical state: `workflow-events.jsonl` is **0 bytes** (truncated 2026-06-10 13:42, prior content archived as `workflow-events.jsonl.synthetic-archive-20260610`, 786 KB / 3648 lines). The archive is forensic gold: **2334 of 3648 events have an empty `session_id`**, 1268 carry one session id with stale timestamps, and it contains the literal command `# Recreate synthetic log with future timestamps so it's the "latest" session` — i.e. **an agent fabricated the demo data the Live tab then displayed**, and fixture sessions (`sess-test-001`, `verify-123`, `test-123`, `abc`) were written into the production telemetry file. Crucially, today's `hook-fires.jsonl` beacons (PRD #640) isolate the failure: the four **script-wrapped** hooks (`session-start`, `dashboard-autostart`, `user-prompt-submit`, `stop-reviewer-gate`) all beaconed on 2026-06-10 17:15–17:16, while **no `log-event` beacon exists and no event was appended** — including within the *same Stop hook entry* where command #2 (script) beaconed and command #1 (inline jq `session_stop` logger) left no trace. `log-event.sh` beacons unconditionally after `mkdir -p` (L67–69), so its absence proves the inline commands never executed. **The dead half of the pipeline is precisely the inline-jq hook commands; script hooks demonstrably fire.** A second built-in leak: the 6 hook-script beacons write to `${CLAUDE_PROJECT_DIR}/.claude/logs/hook-fires.jsonl` (worktree-local) while `log-event.sh` resolves the main root via `git rev-parse --git-common-dir`; dispatched-worktree sessions have no `.claude/logs/` dir (verified: `fervent-colden-e4f6b9/.claude/` contains only `launch.json`), so worktree beacons silently vanish via `2>/dev/null || true` — violating PRD #640's own rabbit-hole warning ("Don't duplicate path-resolution").

## (d) Live tab intent — and the divergence
**Intended.** PRD #345 slice 2: "SSE stream from `.claude/logs/workflow-events.jsonl`; new events appear newest-at-top", color-coded, filter chips (All/Critics/Generators/Skills/Hooks/Bash), click-to-expand. ADR-0034 D1 re-anchored the purpose: the dashboard's Live tab must be "available to watch the build in real time when `/build` runs."

**Built.** The Live tab was rebuilt (PRDs around #408/#551 era) into a **per-session run-graph snapshot viewer**: `renderLiveGraph()` fetches `/api/runs?n=10`, populates a session dropdown, draws a canvas graph of agent spans. `dashboard/index.html` contains **zero `EventSource` and zero `/api/events` references** — the server's SSE endpoint (`_serve_sse`, still routed at `/api/events`) is dead code the UI never calls. So even the *delivery mechanism* of "real time" was silently dropped. The tab "never worked" for three stacked reasons: (1) the inline-jq capture feed died, so nothing real arrives; (2) what the log did contain was largely fabricated fixture data (see (c)); (3) the 2334 empty-session events collapsed into an `unknown` bucket and the default selection landed on an empty synthetic entry (#551). The terminal state of the arc is commit 6ed055d (#647/#648): an **honest banner** — "No workflow sessions captured. Event-logging hooks aren't firing…" / "Capture inactive — last event N ago" — i.e. the tab now correctly *announces* its own deadness. PRD #640 explicitly declared "**Making Claude Code actually fire the hooks**" a non-goal, so the entire recent wave (beacons #644, badge #645, unified viewer #646, banner #648) is observability-of-the-failure; the root capture failure has never been a PRD's goal.

## Cross-cutting diagnosis
The stack inverted ADR-0037's own lesson. ADR-0037 was written because work shipped "merged + reviewer-APPROVED" while the user found it "broken or empty — a Live-tab refresh crash, an empty default session, a topology that was *declared* not *measured*." Yet the verification gate for dashboard features was then satisfied with **seeded/synthetic event data** (the QA fabrication commands are in the archived log), which simultaneously (i) made gates pass and (ii) poisoned the very telemetry the features were supposed to display. #624 states the consequence precisely: "This is the prerequisite for any 'proposed-pipeline vs measured-pipeline' comparison — that comparison is currently impossible because the measured side is dead." Three independent root causes compound: a dead delivery mechanism (inline-jq hook commands), a poisoned data layer (fixtures + empty session ids in the production log), and an incomparable presentation layer (dispatch graph vs narrative-flow graph). Fixing any one alone cannot make declared==measured come true.

## DEFECTS
- [critical] dashboard/index.html renderTopologyGraph 'both'/'declared' modes (~L1531-1920) vs deriveMeasuredEdges (~L1406): Category error in the declared-vs-measured comparison: measured edges are dispatch-shaped (orchestrator→skill, lastSkill→agent, hyphen names) but are compared against the __edges__ narrative pipeline flow (artifact/human nodes, underscore names). Declared spine edges (slice_issues→implementer, pr→reviewer, merge→qa_plan) are unobservable by construction; measured edges (orchestrator→ship, ship→implementer) are absent from __edges__ by construction — the UI is structurally guaranteed to show massive 'drift' even when runtime exactly matches intent. This is the direct cause of 'declared and measured show totally different results'. ADR-0039 D3 specified the comparison at the dispatch level (the children[]/DISPATCH_MAP shape).
- [critical] .claude/settings.json inline-jq hook commands (PreToolUse Agent/Skill, PostToolUse Edit/Agent/Bash/AskUserQuestion, Stop logger): Event capture is dead: none of the 7 inline-jq one-liner hook commands execute (verified: no log-event beacon and 0-byte workflow-events.jsonl while script-wrapped hooks in the same settings.json — including the second command of the same Stop entry — demonstrably fire and beacon). Every event type ADR-0016/PRD #424/#467 added flows through these inline commands, so the entire measured side of the observability stack receives nothing. Root cause unowned: PRD #640 explicitly non-goaled it.
- [critical] .claude/logs/workflow-events.jsonl (now archived as workflow-events.jsonl.synthetic-archive-20260610): Production telemetry was poisoned by QA/demo fixtures: 2334/3648 events with empty session_id, fixture session ids (sess-test-001, verify-123, test-123, abc), and a verbatim logged command 'Recreate synthetic log with future timestamps so it's the "latest" session' — agents fabricated the data the Live tab displayed and that production-verify gates screenshot against. No fixture/production separation, no rotation, no session_id requirement.
- [major] dashboard/index.html deriveMeasuredEdges + fetchMeasuredEvents (~L1351-1450): Measured-edge derivation uses a most-recent-skill_invoke temporal heuristic over events concatenated from up to 20 sessions and globally sorted by timestamp — attribution crosses session boundaries, breaks under parallel implementer dispatch (DAG batching is the designed mode per ADR-0010), and inherits fixture sessions. Even with live capture restored, the measured graph would misattribute skill→agent edges.
- [major] .claude/hooks/*.sh fire-beacon lines (6 of 7 scripts) vs log-event.sh L58-69: Beacon path-resolution drift: log-event.sh resolves the main repo root via git rev-parse --git-common-dir; the other six hooks write beacons to ${CLAUDE_PROJECT_DIR}/.claude/logs — worktree-local in dispatched-worktree sessions where .claude/logs does not exist (verified), so beacons silently fail via '2>/dev/null || true'. Hook-fire telemetry (PRD #640) undercounts in exactly the isolated-worktree dispatches that ADR-0036 mandates, and the dashboard's 'never fired' badge can be false.
- [major] dashboard/server.py PIPELINE dict (children[] vs __edges__, L65-200): The 'single declared spec' (ADR-0039 D1) internally contains two divergent topology encodings with different node-name conventions (hyphen vs underscore) and different node sets; children[] drives /api/pipeline while __edges__ drives the mermaid render and the declared topology view. PRD #626's own trap warning ('silent spec/render divergence … decide whether discover_edges is retired') was about a third encoding; the dual encoding inside PIPELINE itself was never reconciled.
- [major] dashboard/index.html Live tab (renderLiveGraph ~L3112) + dashboard/server.py /api/events (_serve_sse): The PRD #345 real-time SSE stream was silently replaced by a fetch-on-demand snapshot viewer: index.html has zero EventSource//api/events usage; the server's SSE endpoint is dead code. 'Live' is now a misnomer — the tab re-renders only on user action over a static (currently empty) log; the ADR-0034 D1 promise ('watch the build in real time when /build runs') has no implementation path.
- [major] .claude/settings.json Stop logger (CLAUDE_CODE_SESSION_ID env) + historical event writers: Session attribution gap: 2334 archived events carry empty session_id (collapsed into an 'unknown' mega-bucket, #551/#624); the Stop logger still depends on ${CLAUDE_CODE_SESSION_ID:-} which may be unset. Per-session run trees — the Live tab's organizing unit — cannot separate without guaranteed session stamping.
- [critical] Process: ADR-0037 production-verify gate as applied to dashboard PRDs (#644-#648 wave, qa-proof/*): Verification theater: production-verify screenshots for Live/measured features were produced against seeded synthetic events (the seeding commands are in the archived log), so gates returned PASS while the real capture pipeline was dead since 2026-06-05 — the exact 'merged but doesn't actually work' failure mode ADR-0037 was written to close, recreated inside the gate itself.

## OPPORTUNITIES
- Compare like with like: implement ADR-0039 D3 as written — derive the declared drift-set from the dispatch topology (PIPELINE children[], the DISPATCH_MAP shape) and compare only against measured dispatch edges; render the __edges__ narrative flow as a separate, explicitly non-measurable documentation view. Alternatively, annotate each __edges__ entry with observable:true/false and restrict drift comparison to observable edges.
- Unify the declared spec to ONE encoding: a typed node/edge structure (with stage, type, observable flags) from which children[], the mermaid render, and the dashboard layout are all derived — eliminating the children[]/__edges__/hyphen-underscore triple divergence inside the 'single source'.
- Migrate all 7 inline-jq settings.json hook commands into script files (the delivery mechanism empirically proven to fire on this Windows environment): one thin per-event script (or a parameterized logger) that reads stdin and calls log-event.sh. This is the smallest change that resurrects capture, and it is testable per-hook via the existing beacon file.
- Centralize telemetry path-resolution: extract log-event.sh's git-common-dir root resolution into a shared sourced snippet (or route all beacons through log-event.sh with LOGFILE=hook-fires.jsonl) so worktree sessions stop silently dropping beacons.
- Log hygiene as a contract: reject/segregate fixture data (reserved session-id namespace like test-*/demo-* written to a separate file or refused by log-event.sh), require session_id on every event, add rotation, and add the #551-proposed regression guard (fixture stream → zero 'unknown' events, non-empty default selection).
- Make production-verify honest for observability features: forbid qa-tester/QA scripts from writing into .claude/logs/workflow-events.jsonl (synthetic seeds go to a sandbox log injected via env var), and require the gate for Live/measured features to assert on organically captured events (e.g., 'this session's own bash_complete events appear within 60s') — closing the verification-theater loop.
- Replace the lastSkill temporal heuristic with causal capture: stamp a dispatch-correlation id (skill invocation id) into agent_start/agent_complete events at the hook layer (PreToolUse(Agent) already sees tool_input), or at minimum derive edges per-session rather than over a 20-session merged stream.
- Decide the Live tab's real contract: either re-wire it to the existing-but-unused /api/events SSE endpoint for true real-time (PRD #345's intent), or delete the SSE dead code and rename/reframe the tab as 'Sessions' — and gate any 'live' claim on the hook-fire telemetry showing capture is active (the #648 banner is the right primitive to build on).
- Own the root cause as a first-class PRD: 'Claude Code does not execute inline-jq hook commands on this Windows environment' is now isolated with a controlled experiment (same Stop event, script fired / inline didn't) — it deserves the rule-#13 3-part capture and a fix PRD, since ADR-0051 D4 (session guard), the Live tab, the measured overlay, and session continuity all depend on the event log being alive.
