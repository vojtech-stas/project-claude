# topology-ontologist

## SUMMARY
# Topology ontology analysis — why declared ≠ measured is structural, not a capture bug

## The two vocabularies

**DECLARED** (`F:\project_claude\dashboard\server.py` lines 65–201): the `PIPELINE` dict carries TWO independent declarations that disagree with each other:
1. `children[]` (19 dispatch edges, hyphen ids: `orchestrator→build`, `ship→implementer`, …) — a **dispatch hierarchy**.
2. `__edges__` (36 edges, underscore ids, lines 156–200) — an **artifact-flow narrative** (`slice_issues→implementer`, `implementer→pr`, `pr→reviewer`, `reviewer→merge`). Only 8 of the 19 `children[]` edges also exist in `__edges__`.

`__edges__` references exactly **31 node ids** in 3 classes:
- 9 skills: `grill_me, build, ship, to_prd, to_issues, qa_plan, glossary, audit_meta, audit_subagents`
- 10 agents: `prd_critic, adr_critic, slicer, slicer_critic, implementer, reviewer, qa_tester, glossary_critic, backlog_critic, codebase_critic`
- 12 virtual artifact/state nodes with **no PIPELINE dict entry and no file**: `U1, U2, prd_issue, slice_issues, pr, merge, nh, glossary_pr, cap, ptb, bl, capstay` (hardcoded as `VIRTUAL_ARTIFACT_IDS` in `index.html` line 1572).
- **Absent from `__edges__` entirely**: `orchestrator` and `promote-to-backlog` — the only two PIPELINE keys never referenced. `promote-to-backlog` is aliased to the 3-letter artifact id `ptb`.

**MEASURED** (`F:\project_claude\dashboard\index.html`, `deriveMeasuredEdges` lines 1406–1449): consumes exactly **two** event types from `/api/runs` (last 20 sessions of `.claude/logs/workflow-events.jsonl`, merged flat and ts-sorted):
- `skill_invoke{skill}` → emits edge `orchestrator → <skill>` and sets `lastSkill`
- `agent_complete{subagent_type}` → emits edge `<lastSkill|orchestrator> → <subagent_type>`

So the measured grammar can only ever produce two edge shapes: `orchestrator→skill` and `skill→agent`. Measured node vocabulary = `{orchestrator}` ∪ Skill-tool names ∪ `subagent_type` values. Empirically (full archived corpus, 3,648 events): `orchestrator, ship, build, grill-me, to-prd, to-issues, qa-plan, qa-review, implementer, reviewer, prd-critic, slicer, slicer-critic, qa-tester, general-purpose, claude-code-guide` (16 nodes). Events captured but **ignored by topology**: `agent_start` (203), `bash_complete` (3,158 = 87% of corpus), `grill_qa` (62), `session_stop` (8).

## Why they can never match — edge-class arithmetic

Of the 36 declared `__edges__`, classified by endpoint types:
- **20 touch a virtual artifact node** → no event type maps to any artifact; structurally unmeasurable.
- **3 are skill→skill** (`grill_me→ship`, `build→ship`, `ship→to_prd`) → measured grammar emits `orchestrator→skill` for every invocation, never skill→skill; unmeasurable.
- **4 are agent→agent** (`slicer→slicer_critic`, `slicer_critic→slicer` BLOCK, `reviewer→implementer` BLOCK, `codebase_critic→reviewer`) → measured `from` is never an agent; unmeasurable. All BLOCK loops are in this class or the next.
- **1 is agent→skill** (`prd_critic→to_prd` BLOCK) → unmeasurable.
- **8 are skill→agent** (`to_prd→prd_critic`, `to_prd→adr_critic`, `to_issues→slicer`, `qa_plan→qa_tester`, `glossary→glossary_critic`, `ship→codebase_critic`, `audit_meta→reviewer`, `audit_subagents→reviewer`) → the ONLY matchable class.

**Hard ceiling: 8/36 declared edges (22%) can ever turn green, even with perfect capture.** Conversely, every `orchestrator→skill` measured edge — the most common measured edge, one per skill invocation — is **always painted red** ("unexpected drift") because `orchestrator` does not appear in `__edges__` at all. I replayed `deriveMeasuredEdges` over the complete archived corpus against the live `PIPELINE` spec: **25 measured edges → 3 green** (`to-prd→prd-critic`, `to-issues→slicer`, `qa-plan→qa-tester`), **22 red**. Among the red: `ship→implementer` and `ship→reviewer`, which ARE declared in `children[]` — the measured builder produces dispatch-shaped edges (children[]'s ontology) but both-mode compares against `__edges__` (artifact-flow ontology), explicitly "not children[]" per the comment at index.html line 1693.

## What 'both' mode actually compares (d)

Pure string-pair set membership, nothing more. `declaredNamePairs` = `"from::to"` strings from `__edges__` stored in both underscore and hyphen forms (lines 1694–1704); `measuredNamePairs` = `"from::to"` of measured edges (1707–1710). Per declared edge: `measuredNamePairs.has(pairKey)||has(hyphenKey)` → green `#22c55e` else gray (lines 1891–1908). Per measured edge: if not in `declaredNamePairs` (checked in hyphen AND underscore form, lines 1911–1914) → red `#ef4444` overlay edge with dashed-border injected nodes. The underscore/hyphen duality is handled; there is no semantic mapping, no artifact inference, no per-session scoping, no counts, no direction abstraction. The `ptb` alias defeats even string normalization: a real `skill_invoke(skill=promote-to-backlog)` yields node `promote-to-backlog`, whose underscore form `promote_to_backlog` never equals `ptb`, so `ptb→backlog_critic` is permanently gray and `promote-to-backlog→backlog-critic` would be permanently red.

## Measurement-side corruption (compounds the ontology gap)

- `deriveMeasuredEdges` merges events from ≤20 sessions into ONE ts-sorted array; `lastSkill` leaks across session boundaries and across long autonomous phases. After `/ship` Skill-invokes `to-prd`, every later `agent_complete` until the next `skill_invoke` is attributed to `to-prd` — replay shows false reds `to-prd→implementer`, `to-prd→reviewer`, `to-prd→general-purpose`, `ship→prd-critic`.
- **2,337 of 3,648 events (64%) have no `session_id`** (grouped as session "unknown", which is fetched like a real session).
- The PostToolUse·Agent hook captures `tool_response | cut -c1-300` — for the Agent tool that is `{"status":"completed","prompt":"<the INPUT prompt>"}`. The agent's final text (fenced CRITIC trailer `VERDICT/REASON/ROUND`, GENERATOR trailer `RESULT/PR_URL/BRANCH_NAME`) is **structurally absent** from every `agent_complete` event (only 6/201 contain the string "VERDICT", all inside prompt text). Verdict/round/PR data cannot be recovered from existing events.
- The live log `.claude/logs/workflow-events.jsonl` is **0 bytes** (truncated 2026-06-10 13:42); the prior corpus was moved to `workflow-events.jsonl.synthetic-archive-20260610`. That corpus is itself contaminated: sessions `demo-build-7a3f` (25 events) and `demo-ship-2e9c` (14) are fabricated demos with neat timestamps; `abc`, `test-123`, `verify-123`, `manual-test-0609` are 1-event test injections; only `a0aa9ea6…` (1,268 events) is organic, and its only real `skill_invoke` is `qa-review`. The "nice" measured chains (`build→grill-me→ship→to-prd→to-issues→qa-plan`) only ever existed as synthetic data.

## Minimal event vocabulary for declared-expressible measurement (e) + emission points (f)

Existing, keep: `skill_invoke{skill}` (PreToolUse·Skill + `/command` detection in `user-prompt-submit.sh`) — but add an `invoked_by`/skill-stack field (flat `lastSkill` cannot express the declared skill→skill nesting `build→ship→to_prd`); `agent_start/agent_complete{subagent_type}` — add `dispatched_by` and capture the **tail** of `tool_response` (trailer block), not the head.

Missing artifact events, with the exact moment each would be emitted (all are observable `gh` commands — a PostToolUse·Bash matcher on `gh` patterns, or explicit `log-event.sh` calls in the skills, both rule-#12-compatible logging-only):
- `prd_posted{prd}` → node `prd_issue`. Emitted at `/to-prd` publish: `gh issue create --label prd` (`.claude/skills/to-prd/SKILL.md` line 29, post joint-APPROVE).
- `slices_posted{slices[], prd}` → node `slice_issues`. Emitted at `/to-issues` step 5: `gh issue create --label slice` per slice (`.claude/skills/to-issues/SKILL.md` line 20).
- `pr_opened{pr, slice}` → node `pr`. Emitted by implementer step 7: `gh pr create` (`.claude/agents/implementer.md` line 48; `PR_URL` already in GENERATOR trailer).
- `verdict{critic, APPROVE|BLOCK, round, target}` → expresses ALL 5 BLOCK back-edges and the APPROVE edges into `prd_issue/slice_issues/merge/glossary_pr/bl/capstay`. Source: every critic's fenced CRITIC trailer (`VERDICT/REASON/ROUND`) — parse from agent_complete tail, or from the reviewer's `gh pr comment` verdict posts.
- `merged{pr}` → node `merge`. Emitted at reviewer APPROVE: `gh pr merge --squash --auto --delete-branch` (`.claude/agents/reviewer.md` lines 286/317; note `--auto` queues — true merge completes GitHub-side; `MERGE_STATUS` trailer field exists).
- `needs_human{pr}` → node `nh`. Emitted at round-3 escalation: `gh pr edit <PR> --add-label needs-human` (`.claude/agents/reviewer.md` line 301).
- `captured{issue}` → `cap`; `promoted{issue}` → `bl`; `promote_blocked{issue}` → `capstay`. Emitted at `gh issue create --label captured` (any agent) and `/promote-to-backlog`'s `gh issue edit <N> --remove-label captured --add-label backlog` (`.claude/skills/promote-to-backlog/SKILL.md` line 33). Rename `ptb` → the real skill node `promote-to-backlog`.
- `glossary_pr_opened{pr}` → node `glossary_pr`. Emitted at `/glossary`'s PR-open step.
- `U1` = a `user_prompt` event (the `user-prompt-submit.sh` hook already fires but only writes a beacon to `hook-fires.jsonl` + `skill_invoke` for `/commands`); `U2` = `production_verify{PASS|FAIL}` at `/qa-plan`//`/build` step 5 (`PRODUCTION_VERIFY:` trailer already specified per ADR-0037).

Plus invariants: non-empty `session_id` on every event (Stop hook uses `${CLAUDE_CODE_SESSION_ID:-}` env, distinct from the stdin-JSON sid used elsewhere), and per-session derivation (never merge sessions before attribution).

**Bottom line:** declared is an artifact-flow story over 31 ids (12 of them eventless artifacts, plus an aliased skill), measured is a 2-event dispatch grammar over a different 16-id vocabulary rooted at a node (`orchestrator`) that declared doesn't contain, and 'both' compares them by raw string pairs across the wrong one of two mutually inconsistent declared ontologies. Identity (declared == measured) requires (1) one declaration with one id space including orchestrator and the real promote-to-backlog, (2) each declared edge annotated with the event predicate that evidences it, and (3) the ~9 artifact/verdict events above emitted at the listed `gh` touchpoints.

## KEY FACTS

- deriveMeasuredEdges (dashboard/index.html lines 1406-1449) consumes ONLY skill_invoke and agent_complete events and can only emit two edge shapes: orchestrator→skill and lastSkill→agent; agent_start (203 events), bash_complete (3,158 = 87% of corpus), grill_qa (62) and session_stop (8) are captured but never used by topology
- Every measured orchestrator→skill edge is permanently flagged red in 'both' mode because 'orchestrator' does not appear anywhere in __edges__ — this alone guarantees declared ≠ measured under perfect capture
- Edge-class arithmetic over the 36 declared __edges__: 20 touch an eventless artifact node, 3 are skill→skill, 4 are agent→agent, 1 is agent→skill — all 28 structurally inexpressible by the measured grammar; only the 8 skill→agent edges can ever turn green (22% ceiling)
- Empirical replay of deriveMeasuredEdges over the full archived corpus against the live PIPELINE spec: 25 measured edges → 3 green matches (to-prd→prd-critic, to-issues→slicer, qa-plan→qa-tester), 22 red 'unexpected'
- PIPELINE carries two mutually inconsistent declared ontologies: children[] (19 dispatch edges) vs __edges__ (36 artifact-flow edges); only 8 of 19 children[] edges exist in __edges__; both-mode deliberately compares measured against __edges__ ('not children[]', index.html line 1693) even though measured edges are dispatch-shaped — so ship→implementer and ship→reviewer are declared in children[] yet rendered as red drift
- 'both' mode comparison is raw string-pair set membership ('from::to' keys, with underscore/hyphen normalization both ways, index.html lines 1694-1710 and 1891-1914); no semantic mapping, counts, direction abstraction, or session scoping
- The promote-to-backlog skill is declared in __edges__ as artifact alias 'ptb' (also in VIRTUAL_ARTIFACT_IDS, index.html line 1572); skill_invoke(skill=promote-to-backlog) produces node 'promote-to-backlog' whose underscore form never string-matches 'ptb' — permanently unmatchable
- deriveMeasuredEdges merges events from up to 20 sessions into one ts-sorted flat array; lastSkill attribution leaks across sessions and across autonomous phases, producing false red edges (to-prd→implementer, to-prd→general-purpose, ship→prd-critic observed in replay)
- 64% of the real event corpus (2,337/3,648 events) has no session_id and is grouped under session 'unknown', which /api/runs serves like a real session
- The PostToolUse·Agent hook stores tool_response | cut -c1-300, which for the Agent tool is '{"status":"completed","prompt":"<input>"}' — CRITIC trailers (VERDICT/ROUND) and GENERATOR trailers (RESULT/PR_URL) are structurally absent from all 201 agent_complete events; verdict/merge/PR data cannot be recovered from existing telemetry

- qa-review is a real shipped skill and a real measured node but is absent from both the PIPELINE dict and __edges__; subagent_type values general-purpose and claude-code-guide also appear in events with no declared counterpart
- All declared BLOCK loops (prd_critic→to_prd, slicer_critic→slicer, reviewer→implementer) and round-3 escalation (reviewer→nh) are in the structurally-unmeasurable edge classes — the dashboard can never show critic-loop activity as built
- Every missing artifact moment corresponds to an observable gh command: gh issue create --label prd (to-prd SKILL.md L29), gh issue create --label slice (to-issues SKILL.md L20), gh pr create (implementer.md L48), gh pr merge --squash --auto (reviewer.md L286), gh pr edit --add-label needs-human (reviewer.md L301), gh issue edit --remove-label captured --add-label backlog (promote-to-backlog SKILL.md L33)

- [critical] dashboard/server.py PIPELINE.__edges__ + dashboard/index.html VIRTUAL_ARTIFACT_IDS (L1572): 12 declared artifact nodes (U1, U2, prd_issue, slice_issues, pr, merge, nh, glossary_pr, cap, ptb, bl, capstay) have no corresponding event type anywhere in the capture stack; 20 of 36 declared edges touch one and are unmeasurable; combined with skill→skill/agent→agent/agent→skill edge classes, only 8/36 declared edges can ever match
- [critical] dashboard/index.html L1693 (declaredNamePairs 'not children[]') vs PIPELINE children[]: Drift comparison uses the artifact-flow ontology (__edges__) while measured edges are dispatch-shaped (children[]'s ontology); the two declared ontologies disagree (8/19 overlap), so dispatch edges that ARE declared (ship→implementer, ship→reviewer) render as red drift
- [critical] .claude/settings.json PostToolUse·Agent hook (cut -c1-300 of tool_response): agent_complete.output captures '{"status":"completed","prompt":...}' — the input prompt head, not the agent's result tail — so CRITIC trailer VERDICT/ROUND and GENERATOR trailer RESULT/PR_URL are never recorded; verdict and artifact extraction from existing events is impossible

- [major] dashboard/server.py PIPELINE.__edges__ ('ptb', 'cap', 'bl', 'capstay') vs .claude/skills/promote-to-backlog/: The promote-to-backlog skill is declared as 3-letter artifact alias 'ptb'; a real skill_invoke(promote-to-backlog) creates node 'promote-to-backlog' which can never string-match 'ptb' under either underscore or hyphen normalization
- [major] .claude/logs/workflow-events.jsonl (+ .synthetic-archive-20260610): Live log is 0 bytes (truncated 2026-06-10); the archived corpus is contaminated — the only complete measured pipeline chains came from fabricated demo sessions (demo-build-7a3f, demo-ship-2e9c) and test injections; the single organic session logged exactly one skill_invoke (qa-review)

## OPPORTUNITIES
- Collapse to ONE declared ontology: make __edges__ (or a successor edge list) the single declaration; generate children[] from it or delete it; add 'orchestrator' and the real 'promote-to-backlog' as first-class nodes; adopt one canonical id form (kill the underscore/hyphen duality and the ptb/cap/bl/capstay aliases) so event names and node ids are identical strings
- Annotate every declared edge with its evidence predicate (e.g. {from: to_prd, to: prd_critic, evidence: agent_complete[subagent_type=prd-critic, dispatched_by=to-prd]}); compute 'measured' by evaluating predicates over the event stream — declared==measured then holds by construction and drift becomes a per-edge boolean instead of string-pair guesswork
- Emit the ~9 missing artifact/verdict events from the gh touchpoints they already pass through, via a PostToolUse·Bash matcher parsing gh commands (gh issue create --label prd|slice|captured → prd_posted/slices_posted/captured; gh pr create → pr_opened; gh pr merge → merged; gh pr edit --add-label needs-human → needs_human; gh issue edit --remove-label captured --add-label backlog → promoted) — deterministic, logging-only, compatible with rule #12, zero skill-prose changes
- Fix the PostToolUse·Agent hook to capture the TAIL of tool_response (or jq-extract the fenced trailer block) so CRITIC VERDICT/ROUND and GENERATOR RESULT/PR_URL become verdict{critic, approve|block, round} and pr_opened events — this makes all 5 BLOCK back-edges and the merge/escalation edges measurable
- Replace flat lastSkill attribution with per-session, stack-based skill context: add invoked_by/dispatched_by fields at emission time (the PreToolUse hooks have the data) instead of inferring attribution at render time; never merge sessions before deriving edges
- Unify session_id sourcing (stdin JSON everywhere, including the Stop hook) and reject/flag events with empty sid at log-event.sh, eliminating the 64% 'unknown' pollution; purge or quarantine synthetic/demo sessions from any corpus the dashboard reads
- Once the artifact/verdict event vocabulary exists, the Live tab and the measured topology consume the same stream — counts per edge (fire telemetry) become a natural edge weight, and U1/U2 map to user_prompt and production_verify events, closing the loop end-to-end
